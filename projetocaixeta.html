Guia sobre Git e GitHub

O que é o Git

Git é um sistema de controle de versão distribuído usado para rastrear mudanças no código-fonte durante o desenvolvimento de software. Ele permite que múltiplos desenvolvedores trabalhem no mesmo projeto juntos.

O que é o GitHub

GitHub é uma plataforma de hospedagem de código-fonte que utiliza o Git como sistema de versionamento. Ele oferece ferramentas para ajudar, revisão de código, issues, entre outros recursos para desenvolvimento em equipe.

Diferenças entre Git e GitHub

  • Git é uma ferramenta de versionamento local.
  • GitHub é uma plataforma online para hospedar repositórios Git.
  • Você pode usar o Git sem o GitHub, mas o GitHub depende do Git para funcionar.

Principais comandos Linux para usar o Git Bash

cd        # Navegar entre diretórios
ls        # Listar arquivos
pwd       # Exibir o caminho atual
mkdir     # Criar diretórios
touch     # Criar arquivos
rm        # Remover arquivos ou diretórios
clear     # Limpar o terminal

Como criar um repositório no GitHub

  1. Acesse o GitHub e faça login.
  2. Clique em "New repository".
  3. Preencha o nome do repositório e defina se será público ou privado.
  4. Clique em "Create repository".

Como ligar a um projeto existente na máquina

git init
git remote add origin https://github.com/usuario/repositorio.git
git add .
git commit -m "Primeiro commit"
git push -u origin main

Como gerar uma versão com Git

git commit -m "Descrição da nova versão"
git tag -a v1.0 -m "Versão 1.0"

Como enviar da máquina para o GitHub

git push origin main
